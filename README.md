# ⚖️ The Tiebreaker

**An AI decision-support web application.** Describe a decision you're stuck on,
list the options, and tell it what matters — The Tiebreaker turns those
qualitative choices into structured, data-driven outcomes: a **SWOT analysis
matrix** for every option and a **multi-variable weighted priority ranking**.

It runs on the Claude API when a key is present, and ships with a fully
functional **deterministic demo mode** so it works end-to-end with zero setup.

---

## What it does

- **Automated SWOT matrices** — Strengths, Weaknesses, Opportunities, and
  Threats for each option, generated by the LLM.
- **Multi-variable priority list** — every option is scored 0–10 on each
  criterion you care about, then combined into a normalized, weighted total so
  the ranking reflects *your* priorities.
- **A clear verdict** — the single best option with a confidence level and a
  rationale grounded in the criteria that decided it.

## Architecture

```
public/              Static single-page app (HTML + CSS + vanilla JS)
  index.html
  styles.css
  app.js
api/                 FastAPI service — self-contained so it deploys as one
  index.py           Entry point (exports the ASGI `app`; Vercel + local target)
  main.py            API routes; serves the frontend when run locally
  prompts.py         Prompt-optimization layer (system prompt + JSON schema)
  llm.py             LLM bridge: Claude (Structured Outputs) OR deterministic mock
  analysis.py        Weighted priority scoring (pure, testable Python)
vercel.json          Vercel routing + function config
requirements.txt     Dependencies (used by Vercel and the local launchers)
.env.example         Copy to .env to add your Claude API key
run.ps1 / run.sh     One-command local launchers
```

**Request flow:** the browser posts the decision to `/api/analyze`; the
prompt-optimization layer builds a schema-constrained request; Claude (or the
mock) returns SWOT text and per-criterion scores; `analysis.py` computes the
weighted priority ranking; the frontend renders the matrices and rankings.

## The Python prompt-optimization layer

`prompts.py` centralizes every prompt the model sees and pairs it with a
**JSON Schema** enforced via the Claude API's Structured Outputs. This keeps the
model's responses reliable and context-aware: it never returns free-form prose,
only validated structured data that the scoring code can trust. The frozen
system prompt is also prompt-cached to keep repeat requests fast.

## Running it locally

**Windows (PowerShell):**
```powershell
.\run.ps1
```

**macOS / Linux:**
```bash
./run.sh
```

Then open <http://localhost:8000>. The launcher creates a virtual environment,
installs dependencies, and starts the server.

> Requires Python 3.10+. If `py`/`python3` isn't found, install Python from
> <https://python.org> and re-run.

### Enabling real Claude analysis

Demo mode needs no key. For real analysis:

```bash
cp .env.example .env      # then paste your key into .env (project root)
```

Get a key from <https://console.anthropic.com>. The badge in the app header
shows which engine is live.

## Deploying

### Vercel

The repo is set up for Vercel out of the box: the static frontend in `public/`
is served by Vercel's CDN, and `api/index.py` runs the FastAPI app as a Python
serverless function (`vercel.json` routes `/api/*` to it).

1. Push the project to a Git repo (GitHub/GitLab/Bitbucket).
2. In Vercel: **New Project → Import** the repo. No framework preset or build
   command needed — Vercel detects `public/` (static) and `api/` (Python).
3. Add an environment variable **`ANTHROPIC_API_KEY`** (Settings → Environment
   Variables) to enable real Claude analysis. Without it, the deploy still works
   in demo/mock mode.
4. Deploy. That's it.

Or from the CLI: `npm i -g vercel` then `vercel` in the project root.

> **Serverless timeout:** `vercel.json` sets `maxDuration: 60`. A single
> decision analysis usually finishes well within that, but Opus with adaptive
> thinking can occasionally run longer on hard prompts. If you hit timeouts,
> set `CLAUDE_MODEL=claude-sonnet-5` (faster) as an env var, or use a
> long-running host (below) instead.

### Any Python host (Render / Railway / Fly.io / container)

The app is also a single FastAPI service that serves the frontend, so it runs
anywhere Python does — a better fit if you want no request-timeout ceiling:

```bash
pip install -r requirements.txt
cd api && uvicorn index:app --host 0.0.0.0 --port $PORT
```

Set `ANTHROPIC_API_KEY` in the host's environment to enable Claude.

## API

`POST /api/analyze`
```json
{
  "question": "Should I take the startup offer or stay?",
  "options": ["Join the startup", "Stay at current job"],
  "criteria": [{ "name": "Long-term growth", "weight": 5 }]
}
```
Returns the engine used, the weighted results per option (scores + SWOT), and
the recommendation. `GET /api/health` reports whether Claude or the mock is active.
